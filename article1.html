BOFAMET Stealer: Детальный Обзор

Опубликовано: 12.07.2025

Обзор функционала

BOFAMET Stealer — это многофункциональный инструмент, разработанный для компрометации данных из широкого спектра источников на целевой системе. Его возможности охватывают кражу конфиденциальной информации из браузеров, системных данных, документов, криптокошельков, SSH-ключей, а также данных из мессенджеров и игровых клиентов.

1. Поддержка 23+ браузеров

BOFAMET Stealer поддерживает извлечение данных из более чем 23 веб-браузеров, включая популярные Chrome, Edge, Opera, Tor и другие. Извлекаемая информация включает:

  • Пароли
  • Куки-файлы
  • Историю просмотров
  • Данные автозаполнения форм

2. Системная информация

Инструмент собирает обширные данные о скомпрометированной системе, предоставляя полное представление о её конфигурации:

  • Информация об операционной системе (ОС)
  • Характеристики центрального процессора (CPU) и оперативной памяти (RAM)
  • Сетевые интерфейсы, включая MAC-адреса и IP-адреса
  • Геолокационные данные

3. Скрининг файлов

BOFAMET Stealer целенаправленно ищет и извлекает файлы по определённым категориям:

  • Документы: Файлы с расширениями .docx, .pdf, .xlsx.
  • Криптокошельки: Файлы wallet.dat, seed-фразы, а также ключи от Exodus и Atomic Wallet.
  • SSH-ключи: Файлы id_rsa.

4. Мессенджеры

Особое внимание уделяется извлечению сессий из мессенджеров. BOFAMET Stealer способен принудительно завершать процессы Telegram и AyuGram для получения доступа к их сессионным данным.

5. Игровые клиенты

Инструмент также нацелен на конфиденциальные данные, связанные с игровыми платформами, извлекая конфигурационные файлы Steam и Epic Games.

Технические особенности (Билдер и Обфускация)

Для обеспечения скрытности и устойчивости к анализу, BOFAMET Stealer использует продвинутые техники обфускации и защиты.

1. Многослойное шифрование

Процесс обфускации включает несколько этапов шифрования и преобразования исходного кода:

  1. Исходный код
  2. Base16
  3. Маршалинг
  4. Zlib сжатие
  5. Base32
  6. openobf.dll (пользовательская библиотека обфускации)

2. Анти-отладочные механизмы

Для предотвращения анализа и реверс-инжиниринга, BOFAMET Stealer интегрирует анти-отладочные техники:

  • Интеграция с openobf.dll для обнаружения отладчиков.
  • Генерация исполняемых файлов (.exe) без консольного окна с помощью PyInstaller, что затрудняет мониторинг активности.

3. Кастомизация

Билдер предоставляет возможности для настройки создаваемого исполняемого файла:

  • Смена иконки приложения.
  • Автоматическая привязка к управляющему (C2) серверу.

C2-сервер (Система защиты)

Управляющий сервер BOFAMET Stealer также включает механизмы для обеспечения своей безопасности и стабильности.

1. Аутентификация

  • IP-привязка сессий для предотвращения несанкционированного доступа.
  • Блокировка доступа при смене IP-адреса.

2. Защита от сканирования

  • Лимит в 100 попыток входа, после которого панель управления глобально блокируется.
  • Отключение документации API (Swagger/ReDoc) для скрытия структуры API.

3. Блокировка DevTools (JavaScript)

Для предотвращения анализа через инструменты разработчика в браузере, C2-сервер использует JavaScript-код:


setInterval(() => { debugger; }, 100); // Бесконечный цикл отладки, активирует DevTools
document.addEventListener('keydown', e => {
  if (e.ctrlKey && e.shiftKey && (e.key === 'I' || e.key === 'J')) e.preventDefault(); // Блокировка открытия DevTools
});

4. Управление данными

Сервер автоматически удаляет дубликаты данных, используя связку "имя ПК + IP" для идентификации.

Техническая реализация

Основные технологии, используемые в компонентах BOFAMET Stealer:

Компонент Технологии
Сборщик psutil, win32crypt, sqlite3
C2-сервер FastAPI, Uvicorn, Jinja2
Билдер PyInstaller, marshal, zlib

Уязвимости в целевых системах

BOFAMET Stealer эксплуатирует распространённые уязвимости:

  • Незашифрованные SQLite-базы браузеров: Многие браузеры хранят пароли и куки в незашифрованных или слабозащищённых SQLite-базах.
  • Открытый доступ к файлам сессий мессенджеров: Файлы сессий Telegram/AyuGram часто доступны для чтения после принудительного завершения процессов.

cursor.execute("SELECT origin_url, username_value, password_value FROM logins")
